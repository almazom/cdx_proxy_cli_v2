⚡ Quick Start

🚀 One-Liner: Start & Run

bash 
eval "$(cdx proxy --print-env-only)" && codex exec "your prompt here"
🎯 Session Mode

Set up once per shell session

eval "$(cdx proxy --print-env-only)"
codex exec "task 1"
codex exec "task 2"
🔧 Permanent Setup

Auto-start in ~/.zshrc

eval "$(cdx proxy --print-env-only 2>/dev/null)"
🏃 Full Auto Mode

No approvals, full speed

codex exec --full-auto \
  --dangerously-bypass-approvals-and-sandbox \
  "refactor this"

🔑 Recommended Shell Aliases

~/.zshrc 
# Quick proxy setup
alias cdxup='eval "$(cdx proxy --print-env-only)"'

# Full-auto codex
alias codex-auto='codex exec --full-auto --dangerously-bypass-approvals-and-sandbox'

# Status check
alias cdx-status='cdx status --json | jq -r "\"\(.healthy) | \(.auth_count) keys\""'

# Usage: cdxup && codex-auto "your task"

🤖 Codex CLI

codex exec

Run Codex non-interactively.

🎯 Common Patterns

# Full automation (no approvals)
codex exec --full-auto "refactor this function"

# With specific model
codex exec -m gpt-5.3-codex --model-reasoning-effort xhigh "optimize"

# In specific directory
codex exec -C /path/to/project "add tests"

# Danger mode (no sandbox, no approvals)
codex exec --dangerously-bypass-approvals-and-sandbox "deploy"

Flags & Options

FlagDescription
-m, --modelModel (default: gpt-5.3-codex)
-s, --sandboxSandbox: read-only, workspace-write, danger-full-access
-a, --ask-for-approvalApproval: untrusted, on-request, never
-C, --cdWorking directory
--full-auto-a on-request + --sandbox workspace-write
--model-reasoning-effortlow, medium, high, xhigh

🔧 Troubleshooting

⚠️ All Keys Blacklisted

Run cdx reset to reset all keys

ℹ️ Proxy Not Starting

Remove stale PID: rm ~/.codex/_auths/rr_proxy_v2.pid

✅ Check Health

cdx status --json | jq .healthy
